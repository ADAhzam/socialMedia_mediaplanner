# Market & Competitive Insights — Module Reference

This document describes the Insights module: the three insight types, the web-research
flow the skill follows for each, citation and confidence requirements, the mandatory
AM-approval gate, and the exact object shapes consumed by `render/render.js`.

---

## Architecture note

Insights are **not** generated by code. `render/modules/insights.js` only validates that
supplied insights carry a non-empty `sources` array — an empty `sources` produces a soft
warning from `generate()`. The skill (AI) performs the research; the AM approves; the
approved data is set on `plan.insights` before `generate()` is called.

Nothing is fabricated in code. If the skill cannot find reliable data for a field,
it leaves the field populated with a clear placeholder (e.g. `"Data unavailable"`) and
flags it to the AM with a low-confidence note.

## Insight types

### 1. Market Landscape (`plan.insights.marketLandscape`)

Answers: how large is the talent pool, how many candidates are actively looking, how
tight is supply versus demand, and how long are roles typically open?

**Research targets:**

| Field | What to find | Typical sources |
|---|---|---|
| `talentPool` | Total number of people in the region with the required skills/role title | LinkedIn Talent Insights, ONS labour-force surveys, EMSI/Lightcast regional reports |
| `activeSeekers` | Number of candidates actively applying for this role in the target region | Indeed data (published reports), sector-specific job-board traffic estimates |
| `supplyDemand` | Ratio of candidates to open roles, e.g. "8:1" means 8 candidates per vacancy | EMSI/Lightcast, REC (Recruitment & Employment Confederation) sector reports |
| `timeToFill` | Average number of days from posting to hire for this role/region | SHRM benchmarks, LinkedIn Talent Trends, client's own ATS data if supplied |
| `rows` | 2–5 additional metrics with values and strategic implications | Same sources; choose the most actionable metrics for the client's situation |

Each row in `rows` is a three-element array: `[metric, value, implication]`. Example:
```json
["Passive candidates", "90%", "Reach proactively via social — they don't search boards"]
```

**Research flow:**

1. Identify the role family and target region from `plan.roleGroups` and `plan.geoLabel`.
2. Search for labour-market reports covering that role × region combination. Prefer:
   - LinkedIn Talent Insights (if client has access — ask the AM)
   - EMSI/Lightcast regional workforce data
   - ONS or equivalent national statistics office for the target country
   - Published sector-body reports (REC, NHS Employers, etc.)
3. Populate each field from the most recent data available. Always note the publication
   date alongside the citation URL.
4. Construct `rows` with 3–5 metrics that most directly support the campaign rationale.
   Each implication should be actionable (e.g. "Source passively via always-on social").
5. Record all source URLs in the `sources` array.
6. Write a confidence note for the AM (see Confidence tagging below).

### 2. Active vs Passive Split (`plan.insights.activePassive`)

Answers: what percentage of the target talent pool is actively looking versus passively
employed? This data underpins the campaign's social-first strategy.

**Research targets:**

| Field | What to find |
|---|---|
| `passivePct` | Percentage of workers in this role/sector who are passively employed (not actively job-seeking) |
| `activePct` | Percentage actively looking (typically 100 − passivePct) |
| `note` | Short strategic rationale derived from the split, e.g. "Built for the 90%" |

**Typical figures:** LinkedIn's global average is ~70% passive / 30% active for
professional roles. Healthcare, logistics, and social care tend to skew more passive
(85–92%). Software and retail can be more active-heavy during downturns.

**Research flow:**

1. Search for published active/passive split data for the role family and target
   country/region. LinkedIn Talent Trends reports, sector recruitment surveys (e.g.
   NHS People Plan data, Tech Nation reports), and academic labour-market studies are
   useful.
2. If role-specific data is unavailable, use sector-level data and note the caveat in the
   confidence note.
3. State both `passivePct` and `activePct` as integers that sum to 100.
4. Write a short `note` (one sentence) that turns the split into a client-relevant
   message. Example: "Built for the 90% — social campaigns reach passively employed
   candidates that job boards miss."
5. Record source URLs in `sources`.

### 3. Competitive Landscape (`plan.insights.competitive`)

Answers: who are the main employer competitors in this role × region, and how actively
are they recruiting?

**Research targets:**

| Field | What to find |
|---|---|
| `competitors[].name` | Competitor employer name |
| `competitors[].note` | Snapshot of their current recruiting activity (e.g. "Active on Indeed — 14 open nursing roles", "Running Meta video ads — seen in March 2026") |

**Research flow:**

1. Start with `plan.competitors` — these are employer names the AM has already
   identified. Research each one for current activity.
2. Supplement with active-search research:
   - Search Indeed/Reed/LinkedIn Jobs for the role + location and note the top employers
     posting vacancies.
   - Use Meta Ad Library to check whether competitor employers are running recruitment
     ads in the target region.
   - Check LinkedIn company pages for recent "We're hiring" posts.
3. Limit to 6 competitors — the renderer shows a maximum of 6 rows.
4. Write a concise `note` per competitor (one clause, ≤ 15 words) focused on their
   recruiting presence, not their business.
5. Record source URLs in `sources` (one sources array covers all competitors in the
   object).

## Citation requirements

Every insight object must include a `sources` array. Each entry should be a URL
pointing to the specific page or report the data was drawn from. Examples:

```json
"sources": [
  "https://www.linkedin.com/business/talent/blog/talent-strategy/linkedin-talent-trends-2026",
  "https://www.ons.gov.uk/employmentandlabourmarket/peopleinwork/employmentandemployeetypes"
]
```

If a source is a paywalled report (e.g. a client-supplied EMSI report), include the
report name and date instead of a URL: `"EMSI Workforce Intelligence — NW England Nursing, Q1 2026"`.

An empty `sources` array (`[]`) produces a soft warning from `generate()`:

```
Insight "marketLandscape" has no citation/source — verify before sending to client
```

This warning is non-blocking — the file is still written — but the AM must not send the
deck to a client while any insight is uncited.

## Confidence tagging

Present each insight to the AM with a confidence note before approval. Format:

```
Confidence: High | Medium | Low
Basis: <one sentence describing the source and how directly it matches this plan>
```

Examples:

- `Confidence: High | Basis: LinkedIn Talent Insights pulled for RN × Greater Manchester,
  published Q1 2026 — direct match to role and geography.`
- `Confidence: Medium | Basis: ONS national nursing labour-force data extrapolated to
  Manchester region using regional employment share — no direct regional figure available.`
- `Confidence: Low | Basis: Global passive-candidate average from LinkedIn 2024 report —
  no UK healthcare-specific data found; treat as indicative.`

Low-confidence insights should be flagged to the AM explicitly. The AM may decide to:
- Approve with the confidence note added to the slide note field.
- Exclude the insight from the deck entirely.
- Provide their own data from the client's ATS or a proprietary report.

---

## AM-approval gate

Insights must be AM-approved before the deck is sent to a client. This is a mandatory
workflow step, not optional.

**Approval checklist:**

- [ ] Data values are plausible and consistent with the AM's knowledge of the client's
  market.
- [ ] Each insight has at least one citation in its `sources` array.
- [ ] Confidence notes have been reviewed and any Low-confidence insights have been
  discussed.
- [ ] The `note` field on `activePassive` accurately reflects the campaign's strategic
  angle (not just the number).
- [ ] Competitor names in `competitive` have been confirmed as current and relevant —
  the AM verifies no legal, commercial, or relationship issues with naming any listed
  competitor in a client deck.

After approval, set `plan.insights` with the approved objects and run `generate()` to
produce the final deck.

## Exact insight object shapes (renderer field reference)

The following shapes are consumed directly by the slide builders in `render/render.js`.
Field names must match exactly — the renderer reads them without transformation.

### `marketLandscape` → `marketLandscapeSlide`

```json
{
  "talentPool":    "12,000",
  "activeSeekers": "1,200",
  "supplyDemand":  "8:1",
  "timeToFill":    "45 days",
  "rows": [
    ["Passive candidates", "90%", "Reach proactively via social"],
    ["Time-to-fill trend", "+12 days YoY", "Start campaigns earlier in the cycle"]
  ],
  "sources": ["https://example.com/report"]
}
```

- `talentPool`, `activeSeekers`, `supplyDemand`, `timeToFill` — each renders as a stat
  card (large number + label) across the top of the slide.
- `rows` — up to 5 rows; each row is `[metric, value, implication]` (three strings).
  Rendered as a three-column table below the stat cards.
- `sources` — rendered as a small "Sources: …" line at the bottom of the slide.

### `activePassive` → `activePassiveSlide`

```json
{
  "passivePct": 90,
  "activePct":  10,
  "note":       "Built for the 90% — social campaigns reach candidates that job boards miss.",
  "sources":    ["https://example.com/passive-report"]
}
```

- `passivePct` — large number (60pt bold, white) in the left panel.
- `activePct` — large number (40pt bold, navy) in the right panel.
- `note` — italic callout centred below both panels. Omit (empty string or absent) if
  no strategic note is appropriate.
- `sources` — not rendered on this slide (validation only). Citations are recorded for
  the AM approval record.

### `competitive` → `competitiveSlide`

```json
{
  "competitors": [
    { "name": "Bupa",   "note": "Active on Indeed — 14 open nursing roles" },
    { "name": "HC-One", "note": "Running Meta video ads — seen May 2026" }
  ],
  "sources": ["https://www.indeed.com/jobs?q=bupa+nurse", "https://www.facebook.com/ads/library/"]
}
```

- `competitors` — up to 6 entries. Each entry renders as a row: `name` is shown bold
  (navy, left) and `note` in regular weight (grey, right).
- `sources` — rendered as a small "Sources: …" line at the bottom of the slide.
